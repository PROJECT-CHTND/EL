# EL デモシナリオ

企業向けデモで使用するシナリオ集です。

## セットアップ

### 1. Neo4jの起動

```bash
docker-compose up -d neo4j
```

### 2. デモデータの投入

```bash
uv run python scripts/seed_demo_data.py
```

既存データをクリアして投入する場合:
```bash
uv run python scripts/seed_demo_data.py --clear
```

### 3. APIサーバーの起動

```bash
uv run el-api
```

### 4. WebUIにアクセス

http://localhost:8000

## デモシナリオ1: 日報（推奨）

**ターゲット**: 業務報告、進捗管理を行う企業担当者

### ストーリー

> 「1週間前に日報でPRレビューの課題を話していた。  
>  今日また進捗を整理するときに、ELが前の話を覚えていて確認してくれる」

### 事前投入データ

- 「PRレビューのフィードバックに時間がかかる」(1週間前)
- 「PRレビューを24時間以内に完了することを目標にする」(1週間前)

### デモ手順

1. **トピック入力**: 「今日の仕事の進捗を整理したい」

2. **ELの質問に答える**:
   - EL: 「今日は何に取り組みましたか？」
   - User: 「PRレビューを5件やりました」

3. **Wowポイント** ← ここを見せる
   - EL: 「以前、PRレビューのフィードバックに時間がかかるとおっしゃっていましたが、今回はどうでしたか？目標の24時間以内は達成できましたか？」

4. **深掘り会話**:
   - User: 「今回は午前中に集中してやったので、だいぶ早くできました」
   - EL: 「午前中に集中するのが効果的だったんですね。それは良い発見ですね、記録しておきますね」

### 訴求ポイント

✅ 「前に話したことを覚えている」  
✅ 「プロンプトを書かなくても聞き出してくれる」  
✅ 「話すだけで情報が整理・蓄積される」

## デモシナリオ2: 障害振り返り

**ターゲット**: SRE、インフラエンジニア

### ストーリー

> 「先月の障害で『設定変更レビュープロセス導入』を約束した。  
>  今日また障害が起きたとき、ELがその約束を思い出して確認してくれる」

### 事前投入データ

- 「設定ファイルの変更が本番環境に反映されていなかった」(2週間前)
- 「設定変更のレビュープロセスを導入した」(2週間前)

### デモ手順

1. **トピック入力**: 「今日起きた障害について整理したい」

2. **ELの質問に答える**:
   - EL: 「何が起きたか教えてください」
   - User: 「設定ミスで本番環境が止まりました」

3. **Wowポイント** ← ここを見せる
   - EL: 「先月の振り返りで、設定変更のレビュープロセスを導入されましたよね。今回はそのプロセスは通りましたか？」

### 訴求ポイント

✅ 「過去の約束や決定事項を覚えている」  
✅ 「矛盾や変化に気づいて確認してくれる」  
✅ 「再発防止策の実効性を自然にチェック」

## デモでの話し方

### 言うべきこと

- 「プロンプトを書かなくても、話すだけで情報が整理されます」
- 「前に話したことを覚えていて、関連する質問をしてくれます」
- 「矛盾や変化に気づいて、確認してくれます」
- 「ベテラン社員の暗黙知を、組織の資産に変えます」

### 言わないこと

- ❌ 「Knowledge Graphが...」
- ❌ 「Function Callingで...」
- ❌ 「ベクトル検索が...」
- ❌ 「LLMのコンテキスト長が...」

### よくある質問への回答

**Q: ChatGPTと何が違うの？**
> ChatGPTは会話が切れると全部忘れます。ELは話したことを覚えていて、次の会話で活かします。

**Q: どんな情報が保存されるの？**
> 重要な洞察、判断基準、学んだこと、課題などが自動的に記録されます。UIで確認できます。

**Q: セキュリティは大丈夫？**
> データは御社の環境内で保存されます。外部に送信されるのはLLMへの質問のみです。

## トラブルシューティング

### ELが過去の知識を参照しない

1. Neo4jが起動しているか確認
   ```bash
   docker-compose ps
   ```

2. デモデータが投入されているか確認
   ```bash
   uv run python scripts/seed_demo_data.py --verify-only
   ```

### ドメインが正しく認識されない

キーワードを含む発言をすると認識されやすくなります:
- 日報: 「タスク」「プロジェクト」「進捗」「今日」
- 障害: 「障害」「インシデント」「エラー」「復旧」
- レシピ: 「料理」「材料」「調理」
